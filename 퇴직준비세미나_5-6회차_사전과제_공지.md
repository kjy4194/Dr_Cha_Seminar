## 5,6회차 세미나 소개

5,6회차의 주제는 **'연구실이 사라진다 - 개인 연구 인프라'**입니다.

> "파편화된 정보가 연결될 때, 비로소 지혜가 됩니다.  
> PARA 방법론으로 지식을 체계화하면, AI는 당신처럼 생각할 수 있게 됩니다."

5,6회차에서는 PARA 방법론을 적용한 지식 관리 시스템을 구축하고, Obsidian의 양방향 링크를 활용하여 지식 네트워크를 형성합니다.

## 5,6회차 과제의 난이도

이번 과제는 **3,4회차보다 낮지만, 실질적인 시스템 구축이 필요합니다.**

| 비교 항목 | 1회차 | 2회차 | 3,4회차 | 5,6회차 |
|----------|---------|---------|---------|---------|
| 난이도 | ★☆☆☆☆ (1/5) | ★★★☆☆ (3/5) | ★★★★☆ (4/5) | ★★★☆☆ (3/5) |
| 핵심 역량 | 도구 설치 | 프롬프트 엔지니어링 | AI 에이전트 활용 | 지식 관리 시스템 구축 |
| 학습 시간 | 2~4시간 | 4~6시간 | 6~10시간 | 4~6시간 |

3,4회차가 "AI 에이전트를 활용하는 것"이었다면,  
5,6회차는 "**내 지식을 체계화하고 AI가 읽을 수 있도록 정리**하는 것"입니다.

## 왜 5,6회차 과제가 도전적인가?

### 1. 습관의 변화가 필요합니다

단순히 도구를 배우는 것이 아니라, **정보를 정리하는 습관**을 바꿔야 합니다.
- 폴더에 마구 저장하던 방식 → PARA로 분류하는 방식
- 개별 파일로 관리하던 방식 → 양방향 링크로 연결하는 방식
- 기억에 의존하던 방식 → 검색 가능한 지식 베이스 구축

### 2. 자신만의 체계를 설계해야 합니다

정해진 정답이 없습니다. PARA 원칙을 이해하고 **본인의 업무와 관심사에 맞게** 구조를 설계해야 합니다.
- Projects, Areas, Resources, Archive를 어떻게 나눌 것인가?
- 어떤 정보를 노트로 만들 것인가?
- 어떤 노트끼리 연결할 것인가?

### 3. 실제 콘텐츠를 채워야 합니다

빈 폴더만 만들어서는 의미가 없습니다. **실제 업무 지식**을 12개 이상 노트로 정리해야 합니다.
- 진행 중인 프로젝트 정리
- 지속적으로 관리하는 영역 정의
- 참고할 자료와 지식 축적
- 완료된 작업 아카이브

### Obsidian PARA 구조 구축

Obsidian에 PARA 방법론을 적용하여 개인 지식 관리 시스템을 구축합니다.

| 항목 | 설명 |
|------|------|
| PARA 폴더 구조 생성 | Projects, Areas, Resources, Archive 4개 폴더 |
| 노트 12개 이상 작성 | 각 폴더에 최소 3개 이상 (실제 업무/관심사 기반) |
| 양방향 링크 10건 이상 | `[[노트이름]]` 형식으로 노트 간 연결 |
| 그래프 뷰 스크린샷 | 연결된 노드가 보이는 그래프 뷰 캡처 |

### PARA 카테고리 가이드

| 카테고리 | 핵심 질문 | 예시 |
|----------|----------|------|
| **Projects** | "마감일이 있는 진행 중인 일인가?" | 세미나 과제, 연구 보고서, 발표 준비 |
| **Areas** | "끝이 없이 지속적으로 관리해야 하는가?" | 연구 방법론, 프로그래밍 스킬, 건강 관리 |
| **Resources** | "나중에 참고할 자료인가?" | 논문 노트, 튜토리얼, 프롬프트 템플릿 |
| **Archive** | "더 이상 활성화 상태가 아닌가?" | 완료된 프로젝트, 지난 과제 |

### 제출물

1. **폴더 구조 스크린샷** (PARA 4개 폴더가 보이도록)
2. **작성한 노트 목록** (12개 이상, 각 폴더별 노트명)
3. **양방향 링크 예시** (10건 이상, 어떤 노트에서 어떤 노트로 연결했는지)
4. **그래프 뷰 스크린샷** (노트 간 연결 관계가 시각적으로 보이도록)

## 과제 수행 시 참고사항

> 과제는 스스로 해결하시는 것을 권장드립니다.

지식 관리의 핵심은 **"자신만의 체계를 구축하는 것"**입니다.

- 남의 체계를 복사하면 금방 무너집니다
- 시행착오를 거쳐 자신에게 맞는 구조를 찾아야 합니다
- 막히면 YouTube나 Obsidian 커뮤니티를 검색해보세요

**이 과정 자체가 "지식 큐레이션" 역량을 키우는 방법입니다.**

과제 수행 중 해결하기 어려운 문제가 있으시다면, **세미나 질의응답 시간에 함께 해결**해 드리겠습니다. 먼저 본인 힘으로 시도해보시고, 어떤 시도를 했는지 기록해두시면 더욱 효과적인 학습이 됩니다.

## 과제의 의미

5,6회차 과제를 통해 여러분은:

- [ ] 정보를 단순 저장이 아닌 **체계적으로 분류하고 연결**하는 습관 형성
- [ ] 본인의 업무와 관심사를 **지식 베이스로 정리**하는 경험
- [ ] 양방향 링크를 통해 **지식의 네트워크 효과**를 체감
- [ ] **AI가 내 데이터에 접근**할 수 있는 기반 환경 구축

을 얻게 됩니다.

**이것이 바로 "연구실이 사라진다"의 실체입니다.**  
AI가 대신 해주는 것이 아니라, **내가 구축한 지식 체계를 기반으로 AI가 고품질 결과물을 생성**하는 것입니다.

## 마무리

5,6회차는 **진짜 지식 관리**가 시작되는 단계입니다.

지금까지 머릿속에만 있던 지식, 여기저기 흩어져 있던 파일들을 **하나의 시스템으로 통합**합니다.  
그리고 그 시스템을 AI가 읽을 수 있게 연결하면, AI는 당신의 지식을 기반으로 **당신처럼** 문서를 작성할 수 있게 됩니다.

처음에는 PARA 분류가 어색하고, 어떤 노트를 만들어야 할지 막막할 수 있습니다.  
하지만 12개의 노트를 만들고 10개의 링크를 연결하다 보면, **지식이 연결되는 쾌감**을 느끼게 됩니다.

포기하지 마시고, 막히면 AI에게 물어보세요.  
그것 또한 학습입니다.
