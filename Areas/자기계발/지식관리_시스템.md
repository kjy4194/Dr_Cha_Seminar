# 지식관리 시스템

## 개요
Obsidian PARA 기반의 개인 지식관리(PKM) 시스템을 지속적으로 유지하고 발전시키는 영역

## 시스템 현황

### 도구 스택
| 도구 | 용도 | 상태 |
|------|------|------|
| Obsidian | 주 노트 작성 및 연결 | 운영 중 ✅ |
| Git/GitHub | 버전 관리 및 백업 | 운영 중 ✅ |
| Claude AI | AI 보조 작성 | 운영 중 ✅ |

### PARA 구조 현황
| 카테고리 | 설명 | 노트 수 |
|---------|------|---------|
| Projects | 마감일 있는 현재 프로젝트 | 4개 |
| Areas | 지속 관리 영역 | 4개 |
| Resourses | 참고 자료 모음 | 4개 |
| Archive | 완료/비활성 항목 | 4개 |

## 주간 유지보수 루틴

### 매일 (5분)
- [ ] 새 노트 1개 이상 작성 또는 기존 노트 업데이트
- [ ] 캡처한 아이디어를 적절한 PARA 카테고리에 분류

### 매주 (30분)
- [ ] 주간 리뷰: 완료된 Tasks 확인
- [ ] Projects 상태 업데이트
- [ ] 고아 노트(링크 없는 노트) 정리
- [ ] 그래프 뷰로 지식 네트워크 확인

### 매월 (2시간)
- [ ] 전체 vault 리뷰
- [ ] Archive 처리 (완료된 프로젝트 이동)
- [ ] 중복 노트 통합
- [ ] 백업 상태 확인

## 링크 연결 원칙
1. 노트 작성 시 관련 노트 최소 1개 이상 링크
2. [[양방향 링크]]를 활용하여 지식 네트워크 형성
3. 태그(#tag)보다 링크 우선 사용

## 성과 지표
- 총 노트 수: 16개 (목표: 100개)
- 총 링크 수: 12건 (목표: 50건)
- 고아 노트 비율: 0% 유지 목표
