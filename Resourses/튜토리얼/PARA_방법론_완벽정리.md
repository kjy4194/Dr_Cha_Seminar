# PARA 방법론 완벽정리

## 개요
PARA는 Tiago Forte가 개발한 디지털 지식관리 시스템으로,
모든 디지털 정보를 4가지 카테고리로 분류하는 방법론이다.

## PARA란?

| 카테고리 | 영문 | 정의 | 특징 |
|---------|------|------|------|
| **P** | Projects | 마감일 있는 목표들의 시리즈 | 언젠가는 끝남 |
| **A** | Areas | 표준을 유지해야 하는 책임 영역 | 지속적 관리 |
| **R** | Resources | 미래에 유용할 수 있는 주제/관심사 | 참고 자료 |
| **A** | Archive | 다른 3 카테고리의 비활성 항목 | 보관/완료 |

## 각 카테고리 상세

### Projects (프로젝트)
**정의**: 명확한 목표와 마감일이 있는 작업의 시리즈

**특징**:
- 시작과 끝이 있다
- 완료 여부를 명확히 알 수 있다
- 현재 적극적으로 진행 중인 것들

**예시**:
- 논문 작성 (제출 마감: OO월 OO일)
- 연구 보고서 작성
- 세미나 발표자료 준비

**관련 노트**:
- [[퇴직준비세미나_5-6회차_과제]]
- [[지식관리시스템_구축]]
- [[2026_연구보고서_작성]]

### Areas (영역)
**정의**: 시간이 지나도 지속적으로 유지해야 하는 표준이 있는 책임 영역

**특징**:
- 끝이 없다 (지속적)
- 일정 수준 이상을 항상 유지해야 한다
- 역할, 책임, 관심사와 관련됨

**예시**:
- 건강 관리
- 재정 관리
- 연구 역량 유지
- 가족 관계

**관련 노트**:
- [[AI_활용_역량]]
- [[연구방법론]]
- [[자기계발_및_커리어]]
- [[지식관리_시스템]]

### Resources (리소스)
**정의**: 미래에 유용할 수 있는 주제나 관심사

**특징**:
- 현재 당장 사용하지 않아도 됨
- 참고 자료, 학습 자료, 아이디어
- 미래의 프로젝트/영역에 활용 예정

**예시**:
- 프롬프트 모음
- 유용한 링크 모음
- 강의 노트
- 책 요약

**관련 노트**:
- [[프롬프트_엔지니어링_가이드]]
- [[Obsidian_활용법_정리]]
- [[Claude_Code_명령어_모음]]

### Archive (아카이브)
**정의**: 다른 3가지 카테고리에서 비활성화된 항목들

**특징**:
- 완료된 프로젝트
- 더 이상 담당하지 않는 영역
- 관심이 없어진 리소스
- 삭제하기는 아깝지만 현재는 불필요한 것들

**관련 노트**:

## PARA vs 전통적 분류 방법

| 방식 | PARA | 전통 폴더 방식 |
|------|------|--------------|
| 분류 기준 | 실행 가능성 기준 | 주제/토픽 기준 |
| 유연성 | 높음 | 낮음 |
| 검색 효율 | 높음 | 중간 |
| 유지 비용 | 낮음 | 높음 |

## PARA 적용 원칙

1. **최소한의 마찰**: 노트 저장이 쉬울수록 좋다
2. **실행 가능성 기준**: 지금 당장 필요한가? → Projects/Areas
3. **점진적 정리**: 완벽한 분류보다 일단 저장
4. **이동의 자유**: 항목은 카테고리 간 자유롭게 이동 가능
