## 9,10회차 세미나 소개

> "여러 AI 에이전트를 팀처럼 관리하고, 그 경험을 T자형 역량으로 체계화합니다.

9,10회차에서는 Vibe Kanban을 활용하여 멀티 에이전트 병렬 관리를 실습하고, 전체 세미나 학습 내용을 종합하여 T자형 역량을 체계화합니다.

## 왜 9,10회차 과제가 도전적인가?

### 1. 새로운 도구를 익혀야 합니다

Vibe Kanban은 AI 코딩 에이전트를 팀처럼 관리할 수 있는 시각적 프로젝트 관리 도구입니다.
- **Vibe Kanban**: AI 에이전트 관리 대시보드
- **Git Worktree**: 각 태스크별 독립된 작업 환경
- **멀티 에이전트**: Claude Code, Gemini CLI 등 동시 관리

### 2. 프로젝트 관리 능력이 필요합니다

단순히 도구를 쓰는 것이 아니라, **태스크를 설계하고 에이전트에 할당**해야 합니다.
- 어떤 태스크를 만들 것인가?
- 각 태스크의 요구사항을 명확히 정의할 수 있는가?
- 에이전트의 작업 결과를 검토하고 승인할 수 있는가?

### 3. 병렬 작업 관리라는 새로운 패러다임

기존의 순차적 작업에서 벗어나 **여러 에이전트가 동시에 작업**하는 환경을 경험합니다.
- Git 충돌 없이 병렬 작업 가능
- 각 태스크의 진행 상황을 시각적으로 추적
- 완료된 작업을 검토하고 머지하는 프로세스

### 4. 전체 학습을 종합하고 미래를 설계해야 합니다

1회차부터 9회차까지 배운 내용을 한 번에 정리하고 체계화합니다.
- 현재 강점과 약점 파악
- 향후 개발 방향 설정
- 수료 후 지속적 성장을 위한 로드맵 수립

### [9회차] Vibe Kanban 프로젝트 생성 및 관리

`npx vibe-kanban` 명령어로 Vibe Kanban을 실행하고, 최소 3개 이상의 태스크를 등록하여 AI 에이전트로 완료합니다.

| 항목      | 설명                            |
| ------- | ----------------------------- |
| 프로젝트 생성 | Git 저장소 기반 프로젝트 생성            |
| 태스크 등록  | 최소 3개 이상의 태스크 Kanban 보드에 등록   |
| 에이전트 할당 | 1개 이상의 태스크를 AI Agent에 할당하여 완료 |
| 결과 검토   | Diff 확인 및 PR 생성/머지            |

#### 태스크 아이디어 예시

| 태스크 유형 | 설명 | 예상 소요 시간 |
|---------|------|---------------|
|    README 개선 | 프로젝트 문서 작성 및 개선 | 20분 |
|   ️ 유틸리티 함수 | 자주 사용하는 함수 모음 작성 | 25분 |
| ⚙️ 설정 파일 | .editorconfig, .gitignore, package.json 등 | 15분 |
|    데이터 처리 | CSV 파싱, 데이터 변환 함수 | 30분 |
|    테스트 작성 | 기존 코드에 대한 단위 테스트 | 25분 |
|    문서화 | API 문서, 사용 가이드 작성 | 20분 |

## 과제 수행 시 참고사항

> 과제는 스스로 해결하시는 것을 권장드립니다.

9회차의 핵심은 **"AI 에이전트를 팀처럼 관리하고, 병렬로 작업하는 것"**이고, 
10회차의 핵심은 **"객관적 자가 평가와 미래 설계"**입니다.

- 에이전트가 실패하면 프롬프트를 수정해보세요
- 다른 에이전트로 시도해보세요
- Git 충돌이 발생하면 worktree 구조를 확인하세요
- 소감문과 자가 진단은 정답이 없습니다. 솔직하게 작성하세요
- 로드맵은 현실적이고 구체적이어야 합니다

**실패 비용이 거의 제로인 환경에서 빠르게 시도하고 빠르게 학습하는 것이 핵심입니다.**

과제 수행 중 해결하기 어려운 문제가 있으시다면, **세미나 질의응답 시간에 함께 해결**해 드리겠습니다. 먼저 본인 힘으로 시도해보시고, 어떤 시도를 했는지 기록해두시면 더욱 효과적인 학습이 됩니다.
