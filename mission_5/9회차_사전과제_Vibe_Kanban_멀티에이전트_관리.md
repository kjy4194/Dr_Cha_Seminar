## 1. Vibe Kanban 이해하기

### 1.1 Vibe Kanban이란?

Vibe Kanban은 **AI 코딩 에이전트를 팀처럼 관리**할 수 있는 시각적 프로젝트 관리 도구입니다.

```
┌─────────────────────────────────────────────────────────────┐
│  Vibe Kanban 핵심 가치                                      │
├─────────────────────────────────────────────────────────────┤
│  "1명이 10명분 일하기"                                       │
│                                                             │
│  - 여러 AI 에이전트를 동시에 관리                            │
│  - 각 태스크를 독립된 환경(Git worktree)에서 실행            │
│  - Kanban 보드로 진행 상황 시각적 추적                       │
│  - 충돌 없이 병렬 작업 가능                                  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 왜 Vibe Kanban인가?

| 기존 방식          | Vibe Kanban            |
| -------------- | ---------------------- |
| 여러 터미널 창 관리    | 하나의 대시보드에서 모든 태스크 관리   |
| 에이전트 진행 상황 불명확 | 칸반 보드로 실시간 상태 확인       |
| Git 충돌 위험      | 각 태스크별 독립 worktree로 안전 |
| 수동 PR 생성       | 자동 브랜치 생성 및 PR 준비      |

### 1.3 지원 AI 에이전트

| 에이전트            | 설명                        |
| --------------- | ------------------------- |
| **Claude Code** | Anthropic의 공식 AI 코딩 어시스턴트 |

| **Gemini CLI** | Google의 Gemini 기반 에이전트 |

| **Amp** | 강력한 개발 에이전트 |

## 2. Vibe Kanban 설치 및 실행

### 2.1 사전 요구사항

- Node.js 18 이상 설치
- Git 설치 및 설정 완료
- AI 에이전트 설치 (Claude Code 등)

### 2.2 즉시 시작 (설치 불필요)

```bash
# npx로 즉시 실행 (설치 없이 바로 사용)
npx vibe-kanban
```

### 2.3 실행 확인

1. 터미널에서 위 명령어 실행
2. 브라우저가 자동으로 열림 (http://localhost:5173)
3. Vibe Kanban 대시보드 확인

## 3. 프로젝트 생성 및 설정

### 3.1 프로젝트 추가

#### Step 1: 프로젝트 폴더 준비

```bash
# 새 프로젝트 폴더 생성
mkdir vibe-kanban-demo
cd vibe-kanban-demo

# Git 저장소 초기화
git init

# 기본 파일 생성
echo "# Vibe Kanban Demo Project" > README.md
git add .
git commit -m "Initial commit"
```

#### Step 2: Vibe Kanban에서 프로젝트 연결

1. Vibe Kanban 대시보드에서 "+ New Project" 클릭
2. Git 저장소 경로 선택: `/path/to/vibe-kanban-demo`
3. 프로젝트 이름 입력
4. "Add Project" 클릭

### 3.2 프로젝트 설정

```json
// .vibe-kanban/config.json (자동 생성됨)
{
  "project": {
    "name": "vibe-kanban-demo",
    "description": "9회차 사전과제용 데모 프로젝트"
  },
  "agents": {
    "default": "claude",
    "available": ["claude", "amp"]
  },
  "git": {
    "mainBranch": "main",
    "worktreeDir": ".vibe-worktrees"
  }
}
```

## 4. 태스크 등록 및 관리 (최소 3개)

### 4.1 태스크 생성

#### 태스크 1: README 개선

```markdown
## Task: README 파일 개선

### 설명
프로젝트 README.md 파일을 전문적인 형식으로 개선

### 요구사항
1. 프로젝트 소개 섹션 추가
2. 설치 방법 섹션 추가
3. 사용법 섹션 추가
4. 라이선스 정보 추가

### 예상 결과물
- 완성된 README.md 파일
- 마크다운 형식, 이모지 활용
```

#### 태스크 2: 유틸리티 함수 작성

```markdown
## Task: 유틸리티 함수 작성

### 설명
자주 사용하는 문자열 처리 유틸리티 함수 모음 작성

### 요구사항
1. utils/string.js 파일 생성
2. 함수 목록:
   - capitalize(str): 첫 글자 대문자
   - truncate(str, length): 문자열 자르기
   - slugify(str): URL 친화적 문자열 변환
3. JSDoc 주석 포함
4. 단위 테스트 작성 (선택)

### 예상 결과물
- utils/string.js
- (선택) utils/string.test.js
```

#### 태스크 3: 설정 파일 추가

```markdown
## Task: 프로젝트 설정 파일 추가

### 설명
프로젝트 표준화를 위한 설정 파일들 추가

### 요구사항
1. .editorconfig 파일 생성 (에디터 설정 통일)
2. .gitignore 파일 생성/업데이트
3. package.json 초기화 (npm init)

### 예상 결과물
- .editorconfig
- .gitignore
- package.json
```

### 4.2 Vibe Kanban에서 태스크 등록

1. 프로젝트 선택
2. "+ New Task" 버튼 클릭
3. 태스크 제목 입력
4. 설명 및 요구사항 입력
5. "Create Task" 클릭
6. 태스크가 "Todo" 열에 추가됨

### 4.3 태스크 상태

| 상태              | 설명             |
| --------------- | -------------- |
| **Todo**        | 아직 시작하지 않은 태스크 |
| **In Progress** | AI Agent가 작업 중 |
| **Review**      | 작업 완료, 검토 대기   |
| **Done**        | 완료 및 머지됨       |
|                 |                |

## 5. AI Agent에 태스크 할당 및 실행

### 5.1 태스크 할당

1. Todo에 있는 태스크 카드 클릭
2. "Run with Agent" 버튼 클릭
3. 사용할 Agent 선택 (Claude 등)
4. "Start" 클릭

```
┌─────────────────────────────────────────────────────────────┐
│  Task: README 파일 개선                                     │
├─────────────────────────────────────────────────────────────┤
│  Status: In Progress                                        │
│  Agent: Claude Code                                         │
│  Branch: task/readme-improvement                            │
│  Worktree: .vibe-worktrees/task-001                        │
│                                                             │
│  [View Logs]  [Stop Agent]  [View Diff]                    │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 실행 과정 모니터링

Vibe Kanban이 자동으로 수행하는 작업:

```
1. 독립된 Git worktree 생성
   └── .vibe-worktrees/task-001/

2. 새 브랜치 생성
   └── task/readme-improvement

3. AI Agent 실행
   └── Agent가 태스크 수행

4. 변경 사항 커밋
   └── 자동 커밋 (또는 수동)

5. PR 준비
   └── main 브랜치로 머지 준비
```

### 5.3 결과 확인 및 리뷰

1. 태스크가 "Review" 상태로 이동
2. "View Diff" 클릭하여 변경 사항 확인
3. 만족스러우면 "Approve & Merge" 클릭
4. PR 생성 또는 직접 머지

### 5.4 제출물 캡처

다음 화면들을 스크린샷으로 캡처하세요:

- [ ] Kanban 보드 전체 화면 (태스크 3개 이상 표시)
- [ ] AI Agent 실행 중 로그 화면
- [ ] Diff 화면 (변경 사항)
- [ ] PR 생성 화면 또는 머지 완료 화면

## 6. T자형 역량 확장 자가 진단표

### 6.1 T자형 역량이란?

```
               │
     ──────────┼──────────  ← 넓은 범위의 기본 역량 (가로)
               │
               │
               │            ← 깊은 전문성 (세로)
               │
               │
```

- **세로축**: 본인의 핵심 전문 분야 (깊은 전문성)
- **가로축**: AI 도구로 확장한 다양한 영역 (넓은 범위)

### 6.2 자가 진단표 양식

## ❓ 자주 묻는 질문 (FAQ)

### Q1: npx vibe-kanban이 실행되지 않아요.
**A**: 다음을 확인하세요:
- Node.js 버전이 18 이상인지 (`node -v`)
- npm이 최신 버전인지 (`npm -v`)
- 네트워크 연결 상태

### Q2: AI Agent 연결이 안 돼요.
**A**: AI Agent가 별도로 설치되어 있어야 합니다:
- Claude Code: `npm install -g @anthropic-ai/claude-code` 후 인증

### Q3: Git worktree 오류가 발생해요.
**A**: Git 저장소가 올바르게 초기화되었는지 확인하세요:
```bash
git status  # 저장소 상태 확인
git log     # 최소 1개 이상의 커밋이 있어야 함
```

### Q4: PR 생성 없이 직접 머지해도 되나요?
**A**: 네, 개인 프로젝트에서는 직접 머지해도 됩니다. PR 캡처 대신 머지 완료 화면을 캡처하세요.
