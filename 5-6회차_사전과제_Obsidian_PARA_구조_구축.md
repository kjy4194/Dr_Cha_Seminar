## 1. PARA 시스템 이해하기

### 1.1 PARA란?

PARA는 생산성 전문가 **Tiago Forte**가 개발한 정보 조직 방법론입니다. 모든 정보를 **실행 가능성(actionability)**에 따라 4가지로 분류합니다.

```
┌─────────────────────────────────────────────────────────────┐
│  PARA 시스템                                                │
├─────────────────────────────────────────────────────────────┤
│  P - Projects  : 진행 중인 프로젝트 (마감일 있음)            │
│  A - Areas     : 지속적 관심 영역 (마감일 없음, 유지 관리)    │
│  R - Resources : 참고 자료 (언젠가 유용할 정보)              │
│  A - Archive   : 완료/비활성화 항목                          │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 각 카테고리 상세 설명

| 카테고리 | 핵심 질문 | 예시 |
|----------|----------|------|
| **Projects** | "현재 진행 중이고 마감일이 있는가?" | 학위논문, 학회 발표 준비, 프로젝트 보고서 |
| **Areas** | "끝이 없이 지속적으로 관리해야 하는가?" | 연구 방법론, 프로그래밍 스킬, 건강 관리 |
| **Resources** | "나중에 참고할 자료인가?" | 논문 노트, 강의 노트, 튜토리얼, 북마크 |
| **Archive** | "더 이상 활성화 상태가 아닌가?" | 완료된 프로젝트, 지난 학기 수업, 옛 자료 |

### 1.3 분류 판단 플로우차트

```
                    새로운 정보/노트
                         │
                         ▼
              ┌─────────────────────┐
              │ 마감일이 있는가?     │
              └─────────┬───────────┘
                       │
           ┌───────────┴───────────┐
           │ Yes                    │ No
           ▼                        ▼
      ┌─────────┐          ┌─────────────────────┐
      │Projects │          │ 지속적 관리 필요?    │
      └─────────┘          └─────────┬───────────┘
                                    │
                        ┌───────────┴───────────┐
                        │ Yes                    │ No
                        ▼                        ▼
                   ┌─────────┐          ┌─────────────────────┐
                   │ Areas   │          │ 나중에 참고할 것?    │
                   └─────────┘          └─────────┬───────────┘
                                                  │
                                      ┌───────────┴───────────┐
                                      │ Yes                    │ No
                                      ▼                        ▼
                                 ┌───────────┐          ┌───────────┐
                                 │ Resources │          │ Archive   │
                                 └───────────┘          └───────────┘
```

## 2. Obsidian에서 PARA 폴더 구조 생성

### 2.1 기본 폴더 구조

Obsidian Vault에서 다음 폴더 구조를 생성하세요:

```
📁 Your-Vault/
├── 📁 Projects/           # 진행 중인 프로젝트
├── 📁 Areas/              # 지속적 관심 영역
├── 📁 Resources/          # 참고 자료
├── 📁 Archive/            # 완료/비활성화
├── 📁 Templates/          # 템플릿 (선택)
├── 📁 Attachments/        # 첨부파일 (선택)
└── 📁 Daily/              # 일일 노트 (선택)
```

### 2.2 폴더 생성 절차

#### Step 1: Obsidian 실행

1회차 사전과제에서 생성한 Vault를 엽니다.

#### Step 2: 폴더 생성

1. 좌측 파일 탐색기에서 우클릭
2. "New folder" 선택
3. 폴더명 입력: `Projects`
4. 같은 방식으로 `Areas`, `Resources`, `Archive` 생성

#### Step 3: 하위 폴더 구성 (선택사항)

```
📁 Projects/
├── 📁 퇴직준비세미나/
├── 📁 연구프로젝트A/
└── 📁 보고서작성/

📁 Areas/
├── 📁 연구방법론/
├── 📁 프로그래밍/
└── 📁 자기계발/

📁 Resources/
├── 📁 논문노트/
├── 📁 튜토리얼/
└── 📁 도구사용법/

📁 Archive/
├── 📁 2025년-완료/
└── 📁 과거프로젝트/
```

## 4. 양방향 링크 연결하기 (최소 10건)

### 4.1 양방향 링크란?

Obsidian의 핵심 기능으로, `[[노트이름]]` 형식으로 노트를 연결합니다. 연결된 노트는 **백링크(Backlinks)**를 통해 양방향으로 탐색할 수 있습니다.

### 4.2 링크 생성 방법

```markdown
# 링크 생성 문법

## 기본 링크
[[노트이름]]

## 표시 텍스트 변경
[[노트이름|표시할 텍스트]]

## 특정 헤딩으로 링크
[[노트이름#헤딩]]

## 블록 참조
[[노트이름^블록ID]]
```

### 4.3 링크 연결 예시 (10건 이상)

노트를 작성하면서 다음과 같이 링크를 연결하세요:

```markdown
# 링크 연결 체크리스트

1. ☐ Projects/퇴직준비세미나 → [[Obsidian 교육 자료]]
2. ☐ Projects/퇴직준비세미나 → [[PARA 방법론 정리]]
3. ☐ Areas/AI_활용_역량 → [[프롬프트 엔지니어링]]
4. ☐ Areas/AI_활용_역량 → [[Claude Code 사용법]]
5. ☐ Areas/연구방법론 → [[통계 분석 가이드]]
6. ☐ Resources/프롬프트_가이드 → [[프롬프트 예시 모음]]
7. ☐ Resources/Git_명령어 → [[Git 소개와 설치]]
8. ☐ Archive/1회차_완료 → [[3,4회차_사전과제_완료]]
9. ☐ Archive/3,4회차_완료 → [[프롬프트 엔지니어링]]
10. ☐ 자유 연결 1: ______ → ______
11. ☐ 자유 연결 2: ______ → ______
```

### 4.4 백링크 확인

1. 노트 열기
2. 우측 패널에서 "Backlinks" 확인
3. "Linked mentions"와 "Unlinked mentions" 구분

```
┌─────────────────────────────────────────┐
│  Backlinks                              │
├─────────────────────────────────────────┤
│  Linked mentions (3)                    │
│  ├── Projects/퇴직준비세미나.md          │
│  ├── Areas/AI_활용_역량.md              │
│  └── Archive/Phase2_완료.md             │
│                                         │
│  Unlinked mentions (1)                  │
│  └── Resources/학습로그.md              │
└─────────────────────────────────────────┘
```

## 5. 그래프 뷰 활용하기

### 5.1 그래프 뷰 열기

- **단축키**: `Ctrl+G` (Windows) / `Cmd+G` (Mac)
- **메뉴**: 좌측 사이드바에서 그래프 아이콘 클릭

### 5.2 그래프 뷰 이해

```
        ┌──────────────────────────────────────────────┐
        │                 그래프 뷰                     │
        │                                              │
        │       ○ 프로젝트A          ○ 프로젝트B       │
        │        ╲                   ╱                 │
        │         ╲                 ╱                  │
        │          ○───────────────○                   │
        │         ╱ AI_활용_역량  ╲                   │
        │        ╱                   ╲                 │
        │       ○ 프롬프트가이드      ○ Claude_사용법  │
        │                                              │
        │  ○ = 노트                                    │
        │  ─ = 링크                                    │
        └──────────────────────────────────────────────┘
```

### 5.3 그래프 뷰 필터 설정

그래프 뷰에서 톱니바퀴 아이콘을 클릭하여 필터 설정:

- **Filters**
  - 특정 폴더만 표시: `path:Projects/`
  - 특정 태그만 표시: `tag:#중요`
  
- **Display**
  - Arrows: 링크 방향 표시
  - Tags: 태그 노드 표시
  - Attachments: 첨부파일 표시

### 5.4 제출용 스크린샷 가이드

좋은 그래프 뷰 스크린샷을 위한 팁:

1. **충분한 노트가 보이도록**: 12개 이상의 노트가 표시
2. **연결선이 보이도록**: 링크 관계가 시각적으로 확인
3. **폴더별 색상 구분** (선택): 그래프 설정에서 그룹별 색상 지정

## ❓ 자주 묻는 질문 (FAQ)

### Q1: Projects와 Areas 구분이 어려워요.
**A**: "마감일이 있는가?"가 핵심입니다. 
- 특정 날짜까지 끝내야 하는 것 → Projects
- 계속 유지 관리해야 하는 것 → Areas

### Q2: 노트 12개를 무엇으로 채워야 하나요?
**A**: 실제 업무와 관심사를 기반으로 작성하세요:
- 진행 중인 프로젝트 관련 노트
- 배우고 있는 기술/스킬 정리
- 참고 자료 요약
- 완료된 과제 정리

### Q3: 링크를 어떻게 10개나 연결하나요?
**A**: 자연스럽게 연결하세요:
- "이 내용은 [[다른노트]]에서 자세히 설명"
- "관련 자료: [[참고노트1]], [[참고노트2]]"
- "이전 과제 참고: [[Phase2_완료]]"

### Q4: 그래프 뷰가 너무 복잡해요.
**A**: 필터를 활용하세요:
- 특정 폴더만 표시: `path:Projects/`
- 연결된 노트만 표시: 고립된 노트 숨기기 옵션
