# 타임존 / 회의 시간 변환 앱 — 사전과제 따라하기 가이드 (상세)

> **목표**: AI 도구(Claude Artifacts 권장)로 **30분 이내**에 “한 번에 여러 타임존으로 회의 시간을 보여 주는” 웹 앱을 만든다.  
> **과제 연결**: `7-8회차_사전과제_디스포저블_앱_제작.md` 의 Flywheel 보고서·제출 양식에 맞춰 기록한다.

---

## 0. 시작 전에 준비할 것

| 항목 | 내용 |
|------|------|
| **도구** | [Claude.ai](https://claude.ai/) (Artifacts 켜진 계정) 또는 v0.dev, Zoer AI 등 과제 문서의 대안 |
| **타이머** | 스마트폰이나 웹 타이머로 **30분** (과제 목표 시간) |
| **기록용** | 메모장에 **실제 소요 시간** 단계별로 적기 |
| **마음가짐** | “완벽한 세계 지도 앱”이 아니라 **작동하면 OK** |

## 1. 이 앱이 하는 일 (한 문장)

> 사용자가 **날짜 + 시각(한 타임존 기준)** 을 넣으면, **미국 동부·UTC·한국** 등 미리 정해 둔 여러 타임존의 **같은 순간**을 표로 보여 준다.

**왜 디스포저블에 맞나**: 입력/출력이 명확하고, 백엔드 없이 브라우저만으로 구현 가능하다.

---

## 2. 요구사항 정리 (3~5분) — 복사해서 메모해 두기

아래를 그대로 또는 살짝 수정해 “내 앱 스펙”으로 쓴다.

```markdown
## 앱 이름 (예시)
글로벌 회의 시간 변환기

## 목적
한국에서 잡은 회의를 해외 협력자에게 알릴 때, 여러 타임존 시각을 한 번에 확인한다.

## 필수 기능
1. 날짜 선택 (달력 또는 yyyy-mm-dd 입력)
2. 시각 입력 (예: 14:30) + “기준 타임존” 선택 (기본: Asia/Seoul)
3. 변환 결과: 아래 타임존들의 같은 순간 표시
   - Asia/Seoul (KST)
   - UTC
   - America/New_York (미 동부, DST 자동 반영은 브라우저/라이브러리에 맡김)
   - Europe/Berlin (선택)
4. 표 또는 카드 형태로 읽기 쉽게
5. (선택) “복사하기” 버튼으로 이메일/슬랙에 붙일 문장 생성

## 제약
- 별도 서버 없이 프론트엔드만 (React + 브라우저 Date 또는 date-fns-tz / luxon / dayjs timezone 중 하나)
- 모바일에서도 버튼·입력이 크게

## 비필수 (시간 남으면)
- 다크 모드
- 타임존 목록을 사용자가 체크박스로 켜고 끄기
```

## 3. 사용할 도구별 첫 프롬프트 (복사해서 붙여넣기)

### 3-A. Claude Artifacts용 (추천)

Claude 새 대화에서 아래 **전체**를 한 번에 넣는다.

```
역할: 프론트엔드 개발자

다음 요구사항으로 단일 HTML 파일로도 동작하는 React 웹 앱을 만들어 주세요.
(Artifacts로 실행 가능하게, 외부 API 호출 없이)

[앱 이름] 글로벌 회의 시간 변환기

[기능]
1. 날짜: yyyy-mm-dd 입력 필드 (기본값: 오늘)
2. 시각: HH:mm 입력 (24시간제)
3. 기준 타임존: 드롭다운 — 최소한 "Asia/Seoul", "UTC", "America/New_York" 포함
4. "변환" 버튼 클릭 시, 선택한 기준 날짜·시각을 해당 타임존의 로컬 시각으로 해석한 뒤,
   아래 타임존 각각에 대해 **같은 순간**의 날짜·시각을 표로 표시:
   - Asia/Seoul
   - UTC
   - America/New_York
   - Europe/Berlin
5. 타임존 이름 옆에 약어 표기도 (예: KST, UTC, ET는 대략 표기)
6. 잘못된 날짜/시각이면 빨간색으로 에러 메시지
7. UI는 깔끔한 카드 레이아웃, 반응형

[기술]
- React (함수 컴포넌트)
- 시간대 변환은 브라우저 내장 Intl 또는 luxon/dayjs 중 하나로 일관되게 처리
- 한국어 UI 라벨 사용 (날짜, 시각, 기준 타임존, 변환 결과)

[출력]
실행 가능한 전체 코드 한 덩어리로 제공해 주세요.
```

### 3-B. v0.dev용 (짧게)

v0는 토큰이 짧을 수 있어, 위와 동일한 **[기능]~[기술]** 블록만 넣고 마지막에 “Next.js App Router + Tailwind”를 덧붙인다.

## 4. 생성 직후 할 일 (5분)

1. **Artifacts**면 “Open” / 미리보기에서 바로 실행해 본다.  
2. **테스트 시나리오** (메모에 결과만 체크):
   - 오늘 날짜 + `14:00` + 기준 `Asia/Seoul` → 네 줄 모두 논리적으로 맞는지  
   - 기준을 `UTC`로 바꿔도 “같은 순간”이 유지되는지 (숫자만 다른 표현인지)  
3. 이상하면 **5절 수정 프롬프트**로 넘어간다.

## 5. 자주 쓰는 수정 프롬프트 (복사해서 순서대로)

한 번에 하나씩 요청하는 편이 결과가 좋다.

### A. 복사용 문장 추가

```
변환 결과 아래에 "이메일용 한 줄" 텍스트 영역을 추가해 주세요.
형식 예:
"회의: 2026-03-20 14:00 KST (미 동부: 2026-03-20 01:00 ET), UTC: ..."
복사 버튼을 누르면 클립보드에 복사되게 해 주세요.
```

### B. 기본 타임존을 한국으로 고정

```
페이지 로드 시 기준 타임존 드롭다운 기본값을 Asia/Seoul로 해 주세요.
날짜 기본값은 오늘로 유지해 주세요.
```

### C. 모바일에서 입력창이 작다

```
모바일에서 터치하기 쉽게 입력 필드와 버튼 높이를 키우고, 여백을 넓혀 주세요.
```

### D. DST(일광절약) 설명 한 줄

```
결과 표 아래에 작은 글씨로 "미 동부 등은 일광절약에 따라 표기가 달라질 수 있습니다" 한 줄을 넣어 주세요.
```

### E. 에러가 날 때

에러 메시지 전체를 복사해 Claude에:

```
다음 오류가 납니다. 코드를 수정해 주세요.
(여기에 오류 메시지 붙여넣기)
```

## 7. Flywheel 보고서에 넣을 문장 예시

- **필요 발생**: “해외와 회의 잡을 때마다 웹 검색으로 타임존 계산하는 게 번거롭다.”  
- **도구 생성**: “Claude Artifacts에 날짜·시각·기준 TZ → 다중 TZ 표시 요구로 프롬프트.”
