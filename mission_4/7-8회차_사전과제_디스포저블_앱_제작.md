## 1. 디스포저블 앱 개념 이해

### 1.1 디스포저블 앱이란?

**디스포저블 앱(Disposable App)**은 특정 문제 해결을 위해 **빠르게 만들고, 사용하고, 필요 없으면 버리는** 임시 애플리케이션입니다.

```
┌─────────────────────────────────────────────────────────────┐
│  디스포저블 앱 철학                                          │
├─────────────────────────────────────────────────────────────┤
│  "완벽한 앱을 개발하는 대신,                                  │
│   필요할 때 빠르게 만들어 쓰고,                               │
│   목적을 달성하면 버린다."                                    │
│                                                             │
│  - 개발 시간: 10~30분                                        │
│  - 사용 기간: 1회 ~ 수일                                     │
│  - 품질 기준: "작동하면 된다"                                 │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 Flywheel 사이클

```
        ┌────────────┐
        │  1. 필요   │
        │  발생      │
        └─────┬──────┘
              │
              ▼
        ┌────────────┐         ┌────────────┐
        │  4. 폐기/  │ ◄────── │  2. 도구   │
        │  아카이브  │         │  생성      │
        └─────┬──────┘         └─────┬──────┘
              │                       │
              │    ┌────────────┐    │
              └──► │  3. 문제   │ ◄──┘
                   │  해결      │
                   └────────────┘
```

**예시 시나리오**:
1. **필요 발생**: "이 100페이지 PDF 논문의 핵심 내용을 빨리 파악하고 싶다"
2. **도구 생성**: AI에게 "PDF 논문 요약기 앱 만들어줘" (10분)
3. **문제 해결**: 앱으로 논문 5개 요약 완료
4. **폐기/아카이브**: 더 이상 필요 없으면 삭제, 재사용 가능하면 보관

## 3. 앱 제작 실습

### 3.1 앱 아이디어 선택

다음 중 하나 또는 유사한 앱을 선택하세요:

| 앱 유형       | 설명              | 예상 제작 시간 |
| ---------- | --------------- | -------- |
| 📊 데이터 시각화 | 데이터를 차트로 변환     | 15분      |
| 📝 텍스트 분석기 | 글자 수, 키워드 추출    | 10분      |
| 📄 논문 요약기  | PDF/텍스트 요약      | 20분      |
| 🧮 계산기     | 특정 공식 계산        | 10분      |
| 📋 체크리스트   | 커스텀 점검 도구       | 15분      |
| 🔄 포맷 변환기  | CSV↔JSON, 단위 변환 | 15분      |
| ⏰ 타이머/추적기  | 시간 측정, 진행 추적    | 15분      |

### 3.2 Step-by-Step 제작 가이드

#### 예시: 연구 진행률 추적기

##### Step 1: 요구사항 정리 (3분)

```markdown
## 앱 요구사항

### 목적
연구 프로젝트의 각 단계별 진행률을 시각적으로 추적

### 기능
1. 단계 목록 입력 (예: 문헌조사, 실험, 분석, 논문작성)
2. 각 단계별 완료 % 입력
3. 전체 진행률 자동 계산
4. 프로그레스 바 시각화

### 디자인
- 심플하고 깔끔한 UI
- 한눈에 상황 파악 가능
```

##### Step 2: 프롬프트 작성 (2분)

```
다음 요구사항에 맞는 연구 진행률 추적기 웹 앱을 만들어주세요.

[요구사항]
1. 연구 단계 목록을 입력할 수 있어야 합니다
2. 각 단계별로 0-100% 진행률을 슬라이더로 조절합니다
3. 전체 평균 진행률을 자동 계산합니다
4. 각 단계와 전체에 프로그레스 바를 표시합니다
5. 현재 상태를 로컬 스토리지에 저장합니다

[기술 스택]
- React + Tailwind CSS
- 별도 백엔드 없이 프론트엔드만

[디자인]
- 깔끔하고 미니멀한 디자인
- 다크 모드 지원
- 모바일 반응형
```

##### Step 3: AI에 입력 및 결과 확인 (10분)

1. Claude.ai 또는 선택한 도구에 프롬프트 입력
2. 생성된 코드/앱 확인
3. 필요시 수정 요청

```
수정 요청 예시:
"프로그레스 바 색상을 단계별로 다르게 해주세요.
그리고 전체 진행률이 100%가 되면 축하 메시지를 표시해주세요."
```

##### Step 4: 테스트 및 사용 (5분)

1. 앱 실행/미리보기
2. 실제 데이터로 테스트
3. 문제 있으면 수정 요청

##### Step 5: 결과물 저장 (2분)

- URL 복사 (호스팅된 경우)
- 스크린샷 촬영
- 코드 저장 (재사용 대비)

### 3.3 프롬프트 템플릿 모음

#### 데이터 시각화 앱

```
[역할]
데이터 시각화 전문가

[요청]
CSV 데이터를 붙여넣으면 다양한 차트(막대, 선, 원형)로 
시각화해주는 웹 앱을 만들어주세요.

[기능]
1. CSV 데이터 붙여넣기 영역
2. 차트 유형 선택 (막대/선/원형/산점도)
3. X축, Y축 컬럼 선택
4. 차트 이미지 다운로드

[기술]
React + Chart.js 또는 Recharts
```

#### 논문 키워드 추출기

```
[역할]
텍스트 분석 전문가

[요청]
논문 초록이나 본문을 입력하면 
주요 키워드와 빈도를 추출하는 앱을 만들어주세요.

[기능]
1. 텍스트 입력 영역 (대용량 지원)
2. 키워드 추출 (TF-IDF 또는 빈도 기반)
3. 워드 클라우드 시각화
4. 상위 20개 키워드 목록

[기술]
React + 워드 클라우드 라이브러리
```

#### 실험 데이터 통계 계산기

```
[역할]
통계 분석 도구 개발자

[요청]
실험 데이터의 기본 통계량을 계산하는 앱을 만들어주세요.

[기능]
1. 숫자 데이터 입력 (쉼표 또는 줄바꿈 구분)
2. 기본 통계량 계산:
   - 평균, 중앙값, 최빈값
   - 표준편차, 분산
   - 최솟값, 최댓값, 범위
3. 히스토그램 시각화
4. 결과 복사/내보내기

[기술]
React + 통계 라이브러리
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 30분 안에 못 만들겠어요.
**A**: 처음에는 시간이 더 걸릴 수 있습니다. 중요한 것은:
- 완벽하지 않아도 됨 (작동만 하면 OK)
- 복잡한 기능 대신 핵심 기능만
- 실제 소요 시간을 정직하게 기록

### Q2: 어떤 도구를 써야 하나요?
**A**: 가장 쉬운 것부터 시도하세요:
1. Claude Artifacts (Claude.ai에서 바로 사용)
2. v0.dev (Vercel, UI 중심)
3. Nano Banana Pro (이미지/프로토타입)

### Q3: 코딩을 모르는데 가능한가요?
**A**: 네, 가능합니다. AI가 코드를 생성하므로 자연어로 요구사항만 설명하면 됩니다. 다만 기본적인 웹 용어(버튼, 입력창 등)는 알면 좋습니다.

### Q4: 만든 앱을 어떻게 저장하나요?
**A**: 도구에 따라 다릅니다:
- **Claude Artifacts**: 대화 내용 저장, 코드 복사
- **v0.dev**: 공유 URL 제공
- **Replit**: 프로젝트 저장 및 배포
